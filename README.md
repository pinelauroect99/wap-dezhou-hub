# wap-dezhou-hub

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态形式保存，不依赖特定后端服务，便于直接通过浏览器访问或嵌入其他环境。

## 项目简介

这是一个通用型 HTML 页面归档仓库。所有页面均以独立文件形式存放，无框架依赖，无动态内容加载。仓库本身不针对任何特定域名或网站，仅作为页面集合的存储与发布用途。

## 目录结构说明

```
.
├── index.html          # 仓库首页（可选，用于导航或说明）
├── pages/              # 归档页面存放目录
│   ├── example1.html   # 示例页面一
│   ├── example2.html   # 示例页面二
│   └── ...
├── assets/             # 公共资源目录（如 CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md           # 本文件
```

- `pages/`：存放所有归档 HTML 文件，每个文件为一个独立页面。
- `assets/`：存放页面间共享的静态资源，可根据需要组织子目录。

## 页面归档说明

- 所有页面均为纯静态 HTML，兼容主流现代浏览器。
- 页面间互相独立，不共享状态或数据。
- 归档内容不涉及具体业务逻辑或动态数据接口。
- 页面命名建议使用英文小写、短横线分隔，便于路径识别。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有公共资源可放入 `assets/`。
- 更新页面：直接修改对应 HTML 文件，注意保持文件路径与引用资源的一致性。
- 删除页面：移除对应文件，并检查是否有残留引用。
- 不建议在页面内嵌入外部不可控资源（如第三方脚本），以保持归档的稳定性。

## 许可

本仓库内容仅作为页面归档展示，不构成任何形式的授权或使用限制。如需引用或修改，请自行判断合规性。
